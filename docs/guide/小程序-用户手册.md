# 小程序用户手册（全角色）

> 版本口径：基于当前前端代码现状。  
> 环境口径：接口基址为 `https://qip.hauchet.cn/`（见 `src/config/api.ts`）。

## 1. 产品定位与使用边界

- 本小程序当前已形成两类“商城”能力：
  - **服务交易商城（已形成主链）**：围绕检测/认证/计量等服务进行下单与履约。
  - **商品商城（展示态）**：`/pages/mall/index` 可浏览与筛选，但购买/购物车尚未闭环。
- 所有提交类动作统一采用登录守卫：
  - 未登录时提示 `请先登录`；
  - 随后自动跳转 `/pages/auth/login`。

## 2. 快速上手（首日任务清单）

| 角色 | 首日目标 | 推荐路径 | 预期结果 |
| --- | --- | --- | --- |
| 个人用户 | 完成一次服务下单到报告查看 | `登录 -> 服务页选服务 -> 订单创建 -> 样品管理 -> 报告查看/查验` | 形成完整服务交易闭环 |
| 企业用户 | 完成企业资料与资质基础配置 | `登录 -> 我的 -> 企业信息 -> 联系人/开票/收样地址 -> 企业资质` | 企业资料链路可用 |
| 服务机构用户 | 完成一次需求响应 | `登录 -> 需求详情 -> 提交方案 -> 消息跟进` | 可完成接单响应动作 |

## 3. 注册/登录与角色进入

## 3.1 注册路径

- 角色选择页：`/pages/auth/register-role`
  - 个人用户（`individual`）
  - 企业用户（`enterprise`）
  - 服务机构（`agency`）
- 注册表单页：`/pages/auth/register-form`
  - 账号注册：手机号 + 验证码 + 密码。
  - 企业/机构角色会提交主体信息（企业名称、主体类型、资质等）。

## 3.2 登录路径

- 登录页：`/pages/auth/login`
  - 支持短信登录、账号密码登录、微信小程序登录。
- 登录成功后默认进入：`/pages/mine/index`。

## 3.3 个人升级企业（当前前端事实）

- 个人账号可在“我的”页切换账号视角（`personal`/`enterprise`）。
- 企业资料相关页面可直接进入并维护资料。
- 若需要完整主体入驻数据创建，当前主入口仍在注册流程的企业/机构路径。

## 4. 业务主流程

## 4.1 服务交易主链（已联通）

1. 选服务
- 入口页面：`/pages/institution/list`、`/pages/detection/index`、`/pages/certification/index`、`/pages/institution/services`
- 关键能力：筛选服务、进入咨询、带参数跳转下单。

2. 提交委托
- 页面：`/pages/order/create`
- 核心逻辑：若无 `orderId`，先创建直连订单，再提交寄样/委托信息。

3. 订单履约
- 页面：`/pages/order/list`、`/pages/order/detail`
- 能力：查看进度、补录信息、样品操作、评价相关动作。

4. 样品管理
- 页面：`/pages/sample/manage`
- 能力：异常上报、留样/退样登记、流转记录可视化。

5. 报告管理
- 页面：`/pages/report/index`、`/pages/report/detail`、`/pages/report/verify`
- 能力：报告列表、报告详情、防伪查验、下载链接复制。

## 4.2 需求主链（已联通）

1. 发布需求
- 页面：`/pages/demand/publish`
- 能力：保存草稿、提交需求、编辑需求草稿。

2. 需求详情
- 页面：`/pages/demand/detail`
- 能力：查看需求详情与响应建议。

3. 机构响应
- 动作：提交方案（`submitBid`）、关闭需求（`closeDemand`）。
- 说明：服务机构用户当前主要通过需求详情页完成接单响应。

4. 消息触达
- 页面：`/pages/message/index`
- 现状：可作为触达落点，但当前页面为本地消息展示为主。

## 4.3 企业资料主链（已联通）

1. 企业总览
- 页面：`/pages/profile/enterprise`

2. 企业基础资料
- 联系人：`/pages/profile/contacts`
- 开票资料：`/pages/profile/invoices`
- 收样地址：`/pages/profile/addresses`

3. 资质与编号
- 企业资质档案：`/pages/enterprise/certs`
- 服务商编号：`/pages/provider/code`（当前无显式菜单入口）

## 4.4 账号与会员主链（已联通）

- 账号资料与安全：`/pages/settings/index`、`/pages/settings/account`
  - 昵称/头像、用户名、密码、手机号、邮箱、微信绑定、实名认证。
- 会员积分：`/pages/member/vip`
  - 积分余额、规则、流水查询。

## 6. 角色使用指引

## 6.1 个人用户

- 推荐主路线：
  1. 登录后从首页或服务页选择服务。
  2. 提交委托并跟踪订单进度。
  3. 在样品管理补齐流转动作。
  4. 在报告模块查看/查验报告。
- 可选路线：
  - 发布需求并查看机构响应。
  - 在“我的”中管理账号与会员信息。

## 6.2 企业用户

- 推荐主路线：
  1. 登录后先完成企业资料基础配置。
  2. 维护联系人、开票资料、收样地址、资质档案。
  3. 再进入服务交易主链进行批量或持续下单。
- 补充说明：
  - 企业与个人在前端均可走服务下单流程。

## 6.3 服务机构用户

- 推荐主路线：
  1. 登录后进入需求详情。
  2. 提交方案进行需求响应。
  3. 通过消息中心承接后续沟通。
- 补充说明：
  - 当前无独立“机构工作台”页面，接单动作主要在需求详情页完成。

## 7. 当前限制与注意事项

- 商品商城当前为展示态：
  - “加入购物车/立即购买/立即开通”等按钮为占位提示。
- 支付流程未完全开通：
  - 订单详情“立即支付”当前提示建设中。
- 消息中心为轻量形态：
  - 当前前端侧未体现完整后端消息流编排。
- 社区能力为轻闭环：
  - 首页可拉取内容数据，但无独立社区详情页体系。

## 8. 给产品/前端/测试的对齐建议

- 产品侧：
  - 优先补齐“支付链路”和“商品商城交易闭环”。
- 前端侧：
  - 可先补“消息中心后端联通”和“服务商编号入口”。
- 测试侧：
  - 以“服务交易主链 + 需求主链 + 企业资料主链”作为当前核心回归范围。
