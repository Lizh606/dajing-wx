# 现场抽奖活动说明（甲方版）

- 文档版本：v1.0.0
- 适用项目：活动现场“注册即抽奖”小程序功能
- 编制日期：2026-04-23
- 关联接口文档：`docs/api/lottery-openapi.yaml`

## 1. 文档目的

本文档用于向甲方说明“现场抽奖”功能的业务规则、用户体验、现场执行方式与风险控制方案，便于活动前确认口径、活动中执行落地、活动后复盘验收。

## 2. 活动目标

1. 提升现场注册转化率，鼓励到场用户完成注册。
2. 通过“即时开奖 + 现场领奖”增强活动参与感。
3. 形成可追溯的抽奖记录，支撑活动公示与复盘。

## 3. 活动规则（建议最终口径）

1. 参与资格：活动当天完成注册的用户可参与抽奖。
2. 参与次数：每个账号仅可抽奖 1 次。
3. 奖项设置：一等奖 3 份（固定上限），二等奖 20 份（固定上限），三等奖持续发放（不限量）。
4. 兑奖方式：中奖用户凭“领奖编号”到现场工作人员处核销领奖。
5. 活动时间：按 `Asia/Shanghai`（北京时间）自然日口径判定活动当天。

## 4. 用户体验流程

1. 用户进入小程序抽奖页，系统展示当前活动信息与奖项配置。
2. 系统校验用户是否满足抽奖资格（是否当天注册、是否已抽过奖等）。
3. 用户点击“立即抽奖”，转盘动画后展示中奖结果。
4. 页面展示中奖奖项与领奖编号（示例：`L20260423-000327`）。
5. 用户到现场出示领奖编号，工作人员核销后完成发奖。
6. 用户再次进入页面时，可看到历史抽奖结果，避免重复抽奖。

## 5. 现场执行说明（运营侧）

1. 现场物料建议包含：活动规则、奖项数量、领奖地点、截止时间。
2. 核销岗位需按“领奖编号 + 用户信息”进行确认，避免误领。
3. 建议设置“核销台账”或后台核销记录，以便活动后核对。
4. 建议活动结束后导出中奖与核销数据，形成复盘报告。

## 6. 公平性与风控说明

1. 抽奖结果由后端系统生成，前端仅做展示与动画，不参与概率计算。
2. 固定奖项（一等奖/二等奖）按库存控制，避免超发。
3. 同一账号重复请求抽奖时，系统会拦截并返回已抽奖状态。
4. 核销流程支持重复提交防护，避免同一领奖编号重复核销。
5. 所有中奖记录均可追溯（抽奖时间、奖项等级、领奖编号、核销状态）。

## 7. 异常场景与提示口径

1. 已抽奖：提示“当前账号已完成抽奖”。
2. 非活动当日注册：提示“仅限活动当天注册用户参与抽奖”。
3. 活动未开始/已结束：提示“活动尚未开始”或“活动已结束”。
4. 系统繁忙：提示“系统繁忙，请稍后重试”。
5. 核销重复：提示“该领奖编号已核销”。

## 8. 验收标准（甲方可感知）

1. 符合资格用户可完成一次抽奖并获得结果。
2. 不符合资格用户无法抽奖，并收到清晰原因提示。
3. 抽奖结果包含可核销的领奖编号。
4. 页面刷新后可恢复抽奖状态与中奖信息。
5. 核销后状态实时更新，不可重复核销。
6. 奖项发放数量与规则一致，不出现固定奖项超发。

## 9. 交付边界说明

1. 本期交付范围为“抽奖页 + 抽奖结果查询 + 领奖核销能力”。
2. 若需扩展“企业维度限次、黑名单风控、活动多场次管理”，可在后续版本迭代。
3. 若甲方需要对外公示版说明，可基于本文档精简为活动公告文案。

## 10. 甲方确认项（建议会签）

1. 奖项配置与数量是否最终确认。
2. 资格规则是否仅限“活动当天注册”。
3. 领奖截止时间、核销地点、核销责任人是否明确。
4. 异常提示文案是否采用本文默认口径。
5. 活动后数据复盘输出格式（表格字段）是否有指定模板。
