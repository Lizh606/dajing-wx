# AI智享平台小程序 Design System Specification

## 1. Design Vision / 设计愿景
- **设计方向**：以“极简科技”为底座，通过“轻量化呼吸设计”与“适度的色彩点缀”，打造出具备专业可信度且不失人性能量的高效服务对接平台。
- **品牌视觉气质关键词**：极简 (Minimalist)、专业 (Professional)、科技 (Tech-driven)、温暖 (Warm)、呼吸感 (Airy)。
- **用户目标感受**：进入小程序时感到视野开阔、没有压迫感（告别满屏满眼厚重的卡片和蓝底）；浏览信息时层级分明，轻松抓住重点（价格与服务分类）；在交互操作时，能感到高效快捷且富有服务温度（暖色点缀与流畅交互）。

## 2. Brand & Visual Strategy / 品牌与视觉策略
- **品牌定位**：面向B端与C端用户的专业质量服务与检测认证撮合平台，既需要机构级的公信力，也需要消费级产品的易用性。
- **为什么采用“科技冷色 + 温暖点缀”**：冷色系（纯净白、中性灰、点缀科级蓝）传递“精准、严谨、质量认证”的行业属性；而温暖的黄绿、橙黄色作为关键操作点缀，象征“人的服务、快速响应、互助社区”的平台温度，打破传统B端软件冰冷死板的印象。
- **如何平衡“科技感、专业感、服务感、轻量感”**：
  - **科技感与专业感**：克制的色彩排版，高对比度黑白灰体系，通过大面积留白与系统级极简无衬线字体展现。
  - **服务感**：首页金刚区（服务分类）使用饱满的面性彩色图标；社区使用高质量实景照片替代小Icon；使用橙/绿色标签强化用户行动。
  - **轻量感**：坚决摒弃厚重的沉色投影、多重卡片嵌套、满屏蓝色背景。以统一界面基调和去卡片化设计（如浅色弱边框、弥散渐变背景空间）重塑纵向节奏。

## 3. Color System / 色彩系统
> [!IMPORTANT]
> **色彩控制核心原则**：严控蓝色使用比例！仅在头部主视觉（通过柔和逐渐淡出的弥散背景）、关键CTA按钮和强提醒态使用。常规卡片、底色全面回归白与中性极其清淡的灰底。**坚决杜绝“满屏蓝色”、“满屏色块堆砌”。**

- **Primary 主色 (Tech Theme)**
  - `Brand-Blue-500` (#1E61FF)：主品牌色。仅限用于全局主要操作CTA按钮、选中状态色。
  - `Brand-Blue-50` (#F0F5FF)：浅背景主色。用于头部最开始的微弱渐变底色或幽灵按钮选中的打底方案。
- **Accent 点缀色 (Service Warmth)**
  - `Accent-Orange` (#FF8A00)：用于强转化的特权区、价格数字显示、VIP及评分。
  - `Accent-Green` (#10B981)：用作服务达成、认证通过的温和正向状态色彩标签。
- **Neutral 中性色阶 (Typography & Surface)**
  - `N900` (#111827)：Primary Text，大标题、核心文本。
  - `N700` (#374151)：Secondary Text，次级标题、正文文本。
  - `N500` (#6B7280)：Tertiary Text，占位符文字、次要信息说明。
  - `N300` (#E5E7EB)：Border & Line，表单输入框、列表基础分割线。
  - `N100` (#F3F4F6)：Hover Bg / Block Bg，标签基底色，或者特定模块底色。
  - `N50` (#F9FAFB)：App Background，全局默认容器底色。
  - `White` (#FFFFFF)：全部卡片、面板的主容器色。

## 4. Typography System / 字体系统
- **微信小程序适配字体建议**：
  - 中英文/混排默认依赖系统无衬线体 (`system-ui, -apple-system, sans-serif`)。
  - 数字（如金额、统计数字）建议强制采用 `DIN Alternate`, `Inter` 或 `Roboto` 字体族，增强界面里的工业与科技感。
- **字号层级与结构**：(基准尺寸 16px)
  - **Display** (24px, 500/600)：首页醒目主看板数值。
  - **H1 / Title XL** (20px, 500)：大型板块标题、弹窗主标。
  - **H2 / Title** (18px, 500)：常规卡片大标题、列表分类名。
  - **Body 1** (16px, 400)：基础正文、输入框内输入值。
  - **Body 2** (14px, 400)：附加段落信息、次级列表项展现。
  - **Caption** (12px, 400)：状态标签文字、极其弱化的时间戳（搭配 N500颜色）。
- **密集排版场景原则**：
  - 行高 (Line-Height) 通常保持 `1.5 - 1.6`。中文信息密集的地方严禁贴死。
  - 信息必须**主次分明**：任何列表项都要遵循“主信息（500粗黑大），次信息（400偏灰中），附属信息（更灰小字或弱标签）”去执行。

## 5. Spacing & Layout System / 间距与布局系统
- **8pt 栅格系统**：布局体系与内边距必须被 4或8 整除以保持系统节奏规律。
  - `Micro (4px)`：标签内文字距边缘距离、图标与文字靠拢距离。
  - `Small (8px)`：正文字行间距、紧密控件间距。
  - `Medium (16px)`：通用的全局 Screen Margin (屏幕左右留白区)、通用卡片 Padding。
  - `Large (24px/32px)`：截然不同的大楼层 (Section) 之间距，必须保持强留白以实现“呼吸感”。
- **通用结构建议**：
  - **极简降噪流**：减少页面中由于过度强调“分区”而生硬插入的高对比度10px厚灰条，改为更柔和的纯白背景下的间距分隔，甚至1px淡线分割。

## 6. Shape, Elevation & Surface / 圆角、层级与材质
- **圆角体系 (Radius)**：
  - `Radius-SM (4px)`：小巧的信息标签(Badge)，多选框等。
  - `Radius-MD (8px)`：常规操作按键、搜索框、配图图片倒角。
  - `Radius-LG (12px/16px)`：内容大卡片(Card)、底部弹窗(BottomSheet)、Banner主视觉外框。
- **阴影与层级分类 (Elevation)**：
  - **Level 0 (Flat/基础)**：普通浏览卡片及内容框。原则：纯白背景`White` + 极微弱边缘线 `border-N300(E5E7EB)`，**不带明显阴影，极致扁平**。
  - **Level 1 (Hover/Click)**：极弱长影 `0 4px 14px rgba(0,0,0,0.03)`，作为部分核心卡片选中态。
  - **Level 2 (Pop/Modal)**：深层阴影 `0 10px 40px rgba(0,0,0,0.08)`，作用于彻底盖在上一层界面的下拉菜单或者Dialog。
- **表面材质应用 (Gradient / Surface)**：
  - 使用**渐变/弥散背景映射**替代以往生硬的顶部实色Header。
  - **规避错误**：“满屏深灰色/浅灰色卡片”会导致非常严重的沉闷感和“脏乱”感。尽量让界面的物理面（背景与卡片背景）融为了透明白。

## 7. Iconography / 图标系统
- **首页主功能分类 (金刚区) Icon**：
  - **风格**：扁平化(Flat)，采用轻度面性或微立体光泽，造型要丰满、饱满。不能是细若游丝的细线图标。
  - **色彩区隔**：根据不同服务大类（如质量检测、需求撮合）赋予蓝、青、温暖橘、黄绿的基调色。不依赖大量单一的蓝色系。
- **服务页与列表操作 Icon**：
  - 退化为主流线性 (Line) 图标（建议2px或1.5px端点圆滑绘制描边），使用纯正的中灰色N700。
  - 优先级必须**大幅弱化**，仅作为文本的引导辅助。
- **我的页快捷入口 Icon**：
  - 也是标准线性纯色套系，视觉不能喧宾夺主，和整体清爽的面板保持同呼吸。

## 8. Imagery / 插画与图片规范
- **首页 Banner 主视觉**：抛弃繁复的大字报运营图片设计，改成含有轻柔几何渐波纹/弥散光影的“简约背景配一句科技感文案和搜索区”的无界融合视觉结构。
- **质量社区配图规范**：
  - **强制要求**：采用高清、具备质感的现场/服务/科研/专业人员等的“实景真实照片 (Photography)”。不准再使用简单的矢量小插图配资讯，这样显得极度廉价。
  - 裁切与处理：推荐固定比例（如 `16:9`、`4:3` 封面视图），四角加上不大于 8px 的内敛圆角。不出现极重水滴状圆角或异形剪纸。
- **搭配规律**：该纯图文的地方大胆释放图片魅力，非内容引导的流程则收起图，走纯文字与数据层展。

## 9. Component Principles / 组件设计原则
### Button / 按钮体系
- **主按钮 (Primary)**：唯一行动点，使用 `bg-Primary-500`。
- **次级按钮 (Secondary)**：用于较多并列选项，采用灰边框去面性设计 (`border N300 text-N700 bg-White`)。
- **操作层级痛点规避**：不可将详情列表中所有动作都做成强按钮，大幅缩减页面里并存的按钮数量，次要操作必须降级为文字链接、Tag形式或收纳。

### Tab / 一二级分类切换
- **严禁样式**：不要在一级和二级Tab上堆叠出厚重的“外包大色块框”或“多重满变色胶囊”。
- **规范样式**：
  - 一级Tab采用“黑体大字 (N900) + 底部短粗科技蓝下划线 (如宽16px/高3px的横条)”。
  - 二级分类Tag用“极淡浅灰底 N100 + 圆角小胶囊”设计，选中态才变为较醒目色。

### Search & Select / 搜索表单
- **Search Bar**：抛开粗硬描边，改用胶囊圆弧形 (`Radius-Full`)，填充极淡底层背景(`N50 / N100`)。
- **Filter Chips (筛选签)**：小尺寸、浅色背景的点击型Tag，体现高级感的展开交互机制，而不是堆砌所有的单选框。

### Card / 核心卡片逻辑
- 摒弃大片灰色的信息堆叠区。
- 左图右文：图片在左侧规矩对齐；标题在右上方使用大黑体。
- **信息强化**：使用 `Accent-Orange` 去强化展示价格或者费用；服务标签降低信息量、收缩字号大小到 `12px` 并改用含有一点品牌底色的极浅白蓝或者极浅白绿框，增加阅读清晰度。

### Bottom Navigation / 底部导航
- 背景纯白，顶侧带一条极淡的 `1px #F3F4F6` 灰线，不用顶部投射浓烈黑影。图标未选中时使用中灰线性，选中时转化为有温度的主色渐变或者蓝色面性图标。

## 10. Page-Level Design Guidelines / 页面级设计规范
### 首页重构
- **页面目标**：首屏抓眼、专业可信且充满呼吸动线。
- **顶部沉浸重构**：打通传统的硬边缘Header顶图。允许采用柔美的浅科技蓝弥散背景 `linear-gradient(to bottom, #F0F5FF, #FFFFFF)` 向下淡出，自然接入内容白底，产生“无边框延伸”高级感。
- **版面减负**：需求大厅卡片不准用重灰块打底，必须切换为“高反白”+“极少跳色彩色小标签点缀”模式，让视野通透。

### 服务页重构
- **页面目标**：高效找寻，直击价格和质量优势，去除眼花缭乱的视觉障碍。
- **蓝色与按钮“减肥”**：大幅削减原本过多的蓝色块背景和每一行强行加塞的操作按钮。将服务直接变成可点击的“白块区域主体”。
- **搜索与导航**：分类不堆叠大圆角按钮，而是改成简单的横向吸顶文本 Tab 切换滑动流。

### 我的页重构
- **页面目标**：收纳管理与降噪，提供“安静服务支持”的氛围体验。
- **最核心去卡片重叠区**：原顶部的极其厚重的深蓝色资产块和半身卡片叠加形式应抛弃。取而代之的是浅色调、或者极其微弱色域的浅彩色背景下，直接将用户信息和平坦的概览信息摆上。
- **模块清淡分界**：下方的各个业务管理入口不再需要重阴影外壳护体，全部平铺使用白底并以优雅平滑的水平 1px 极弱分割线进行区块引导。

## 11. Interaction Principles / 交互原则
- **操作反馈**：在金刚区、卡片等地方应附带轻交互按压态。用户轻按会发生透明度 `opacity 0.7` 或缩放 `scale 0.98` 改变，不一定非得增加点击阴影。
- **Tab 切换效果**：下划线的游标需要做阻尼感水平平移动画，体现科技的顺滑，而不是直接硬切闪现。
- **体现轻与快**：坚决落实“骨架屏(Skeleton)”方案，放弃所有阻塞感长达数秒的“Loading菊花图标”强接管界面，业务数据再多也不能让用户感觉在死等一个界面反应。

## 12. Accessibility & Usability / 可访问性与易用性
- **点击热区**：作为移动端应用，不管视觉层面按钮拉得多简约、多小、多幽灵，**实际隐形点击触发热区禁止低于 44*44 pt**。
- **文字识别度**：再多追求轻度的弱文本也不许将 #E5E7EB 用作文字阅读。请死死卡住最小灰度下限用 #9CA3AF (N400) 或 #6B7280 (N500)。
- **色觉障碍兼容**：标签传递核心状态错误（例如打回调）、通过审核等不能简单地仅用一小片红色或者绿色块圆点，无论颜色怎么变，必须带上对应的文字表述“待处理 / 不通过”。

## 13. Tone of UI Copy / 界面文案语气
- **语调基准**：严谨不失人情味，顾问式服务关怀口吻。
- **避免机器感**：避免提示系统错误的干瘪“ERROR：1090x”；引导使用“哎呀，网络出了点小差错”并给出售后/重试按钮入口。
- **动作按钮**：必须动宾结构对应意图，例如写明“立即发布需求”而不是干巴巴的一个字“走”。

## 15. Design QA Checklist / 设计验收清单
在发版或者UI设计师走查界面时，请逐一核对以下 5 条“极简呼吸”红线：
- [ ] **色彩泛滥度走查**：检查这一屏是否有过多无意义的颜色背景？主色蓝使用比例是否得到了极度克制？
- [ ] **按钮拥挤度走查**：列表和商品展示中是否因为操作过多而塞满大量冗余线框按钮？是否将优先级降解处理正确？
- [ ] **阴影与多重框走查**：所有普通卡片是否遵循不加任何脏旧深色投影的原则，是否已经拆解了卡片包裹卡片的重复嵌套地狱局面？
- [ ] **Tab排板走查**：切换分类是否换成了极简平铺文字下划线条？
- [ ] **社区资讯排版走查**：新闻公告里的文章配图是否使用了真实摄影质感的封面？是否有强列图文版式的左近对齐秩序？

---

> [!TIP]
> **终极设计改造方略回顾**
> 
> *   **高保真视觉关键词**：呼吸感 (Airy)・高公信力 (Trustworthy)・冷峻科技加暖阳点缀 (Tech-Cold with Touch of Warmth)。
> *   **适合参考的设计风格方向**：Vercel、Stripe 官网级别的清透 B2B SaaS 质感；或者苹果 App Store 中极少阴影原生应用体验（少即是多 / Less is more）。
> *   **首页 / 服务页 / 我的页 三页统一改版核心动作**：
>     1.  **拔掉复杂形体**：去掉深色卡片嵌套和满屏满盖色的区块切割，用绝对的纯白、宽广的内边距来实现空间分割自然感。
>     2.  **视觉权重还给内容**：服务页将抢戏的系统框或乱色块剥除，直接让用户第一秒聚焦**价格、实景高清物料和分类标签上**。不相关的操作均统一隐退。
>     3.  **头部降噪破局**：“我的”与“首页”抛弃顶高突兀隔断大色块，用背景弥散羽化科技光感取代硬线条割裂感。让从眼进入界面的感受像水流一样无缝自然向下滑动。
